# Easy Dashboard

`Easy Dashboard` — это плагин для Obsidian, который генерирует красивую домашнюю страницу с быстрыми действиями, недавними заметками, папками, тегами и вспомогательными экранами.

Он упаковывает всю логику в готовый плагин, чтобы людям не приходилось вручную копировать длинные блоки `dataviewjs`.

## Возможности

- Генерирует домашнюю страницу-дэшборд
- Автоматически открывает дэшборд при запуске Obsidian
- Показывает приветствие в зависимости от времени
- Добавляет быстрый поиск
- Добавляет кнопки для создания заметки, открытия inbox и карты
- Показывает недавние заметки в виде карточек
- Показывает папки в виде кнопок
- Скрывает системные папки из списка папок
- Для папок внутри `Sort/...` показывает только последнее имя папки
- Показывает теги в виде кнопок
- Генерирует отдельные страницы для просмотра папок и тегов
- Добавляет команды для открытия и обновления дэшборда
- При желании всё ещё может подключаться к плагину `Homepage`

## Требования

- `Dataview` обязателен
- `Templater` опционален
- `Homepage` опционален и больше не нужен для автозапуска дэшборда

## Установка

### Установка вручную

1. Скачайте `manifest.json`, `main.js` и `versions.json`
2. Создайте папку `easy-dashboard` внутри `.obsidian/plugins/`
3. Поместите эти файлы в папку плагина
4. Включите плагин в разделе community plugins в Obsidian

### Установка через GitHub release

1. Скачайте файлы последнего релиза
2. Распакуйте архив
3. Поместите `manifest.json`, `main.js` и `versions.json` в `.obsidian/plugins/easy-dashboard/`
4. Включите плагин в Obsidian

## Настройка

1. Включите `Dataview`
2. Откройте `Settings -> Easy Dashboard`
3. Укажите пути к заметкам, папке inbox, шаблону и при необходимости заметке карты
4. Убедитесь, что включена опция `Open on startup`, если вы хотите использовать плагин вместо `Homepage`
5. Нажмите `Generate`

## Команды

- `Generate or update dashboard files`
- `Open dashboard`
- `Connect dashboard to Homepage plugin`

## Примечания

- Плагин генерирует обычные markdown-заметки с `dataviewjs`, поэтому пользователи всё ещё могут вручную менять внешний вид после установки.
- Значения путей по умолчанию в этом репозитории отражают личную структуру vault Анны. После установки другим пользователям стоит открыть настройки и подстроить пути под свой vault.

## Чеклист релиза

1. Обновите версии в `manifest.json`, `package.json` и `versions.json`
2. Соберите zip-архив с файлами:
   - `manifest.json`
   - `main.js`
   - `versions.json`
3. Опубликуйте GitHub release с тегом, совпадающим с версией в `manifest.json`
